Intro

Hello! I’m Gregory, a seasoned Data Analyst with five years of experience in designing and executing scalable data solutions across various industries. I specialize in leveraging cloud platforms, especially AWS and Microsoft Power Platform, to build robust data integration, validation, and automated reporting solutions. My expertise spans data modeling, ETL pipelines, and database management, with proficiency in SQL, Python, and Power BI, enabling me to deliver powerful insights that drive business performance.

Throughout my career, I’ve taken on roles in data engineering, Program Manager, and security analytics, where I’ve led impactful projects like designing AWS-based data warehouses, streamlining big data processing with tools like Snowflake and Amazon glue, and optimizing real-time reporting through interactive Power BI dashboards. My work has led to tangible improvements in data accessibility, reporting accuracy, and decision-making efficiency, benefiting teams across marketing, sales, and operations.

I hold an MSc in Data Engineering from Dublin Business School and a BSc in Mechanical Engineering from the University of Debrecen. My commitment to continuous learning is backed by certifications in Master Data Engineering (MDE) and Project Management (PMP). I’m excited to share my journey and insights with you—feel free to explore my projects, and let’s connect!

NYC Payroll

The NYC Payroll Data Integration Project aims to create a unified Data Analytics platform for analyzing and sharing payroll data across all New York City agencies. Key objectives include: Financial Analysis: To examine how resources are allocated, focusing on budget portions dedicated to salaries and overtime.

Public Transparency: To provide public access, allowing transparency around the City's payroll and overtime expenses. The project involves designing a data warehouse, building an automated ETL pipeline, and developing aggregate tables for streamlined analysis. Data quality and consistency are prioritized, and a public user role with limited access will enable safe data transparency. Comprehensive documentation and a cloud-hosted, version-controlled repository will support collaboration and reproducibility.

Zipco

The Zipco Foods Data Engineering Project aims to centralize and optimize the company’s fragmented sales and inventory data. Due to data being scattered across multiple CSV files, Zipco faces inefficiencies in accessing real-time insights and maintaining data integrity.

The project’s goal is to build a scalable ETL pipeline that consolidates this data into a unified database, improving data accessibility, accuracy, and operational efficiency. Key steps include automating data ingestion, cleansing, transformation, and storing processed data for analysis, enabling real-time dashboards for enhanced decision-making.

GoalBet

GoalBet ETL Pipeline Project: GoalBet, a sports data analytics company, seeks to develop an end-to-end ETL pipeline to centralize sports data from multiple sources for predictive modeling, insights, and betting analysis. Key goals include improving data handling efficiency and speeding up query operations for faster, cost-effective insights.

Project Highlights Objective: Automate data extraction, transformation, and loading into a Postgres database. Benefits: Enhanced query performance, operational efficiency, and cost savings. Technologies: Python, SQL, Postgres, Visual Studio Code, GitHub. This solution will streamline GoalBet’s data processing, enabling quicker responses to market shifts and reducing infrastructure costs over time.
